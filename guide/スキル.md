# エージェント・スキル (Skills)

**スキル**は、エージェントの機能を拡張するためのオープンな標準規格です。特定のタスクに対する指示やベストプラクティスをパッケージ化することで、エージェントの振る舞いをカスタマイズできます。

---

## 🧩 スキルとは？

スキルは、エージェントが「何をすべきか」だけでなく「**どのように取り組むべきか**」を定義した再利用可能なナレッジパッケージです。

- **アプローチの指示**: 特定のタスクに対する最適な進め方。
- **規約とベストプラクティス**: プロジェクト固有のルールや命名規則。
- **リソース**: 補助スクリプトやテンプレート。

## 📁 スキルの配置場所

スキルは以下のいずれかのディレクトリに配置します。

| スコープ               | パス                            | 用途                                     |
| :--------------------- | :------------------------------ | :--------------------------------------- |
| **ワークスペース固有** | `<root>/.agent/skills/`         | 特定のプロジェクト専用のルール           |
| **グローバル**         | `~/.gemini/antigravity/skills/` | すべてのプロジェクトで共通利用するツール |

---

## 🛠️ スキルの作成方法

1. スキルディレクトリ内に新しいフォルダを作成します。
2. その中に `SKILL.md` ファイルを作成します。

### 📄 SKILL.md の構造

ファイルの上部には必ず **YAML フロントマター** を記述してください。

```markdown
---
name: my-skill
description: Python の pytest を使用したユニットテスト作成を支援します。
---

# スキル名

ここに詳細な指示を記述します。

## 使用タイミング

- Python の新機能を追加したとき
- リファクタリング後の検証が必要なとき
```

### 📂 推奨されるフォルダ構成

`SKILL.md` 以外にも、必要に応じて以下のリソースを含めることができます。

```text
.agent/skills/my-skill/
├── SKILL.md      # メインの指示書 (必須)
├── scripts/       # 補助スクリプト
├── examples/      # リファレンスコード
└── resources/     # テンプレート等
```

---

## 🚀 エージェントによる活用サイクル

エージェントは以下の流れでスキルを「発見」し「実行」します。

1. **発見 (Discovery)**: 会話開始時、利用可能なスキルの名前と説明をスキャンします。
2. **活性化 (Activation)**: タスクに関連があると判断した場合、`SKILL.md` の詳細を読み込みます。
3. **実行 (Execution)**: 指示に基づき、一貫性のあるアウトプットを生成します。

> [!TIP]
> エージェントに明示的に「@スキル名 を使って」と指示することで、確実に特定のスキルを適用させることができます。

---

## 💡 ベストプラクティス

- **責務を絞る**: 1つのスキルに多くの役割を持たせず、単一のタスクに特化させます。
- **説明を明確に**: `description` はエージェントがスキルを選ぶ際の唯一の判断材料です。具体的かつ簡潔に記述しましょう。
- **ブラックボックス化**: 補助スクリプトがある場合、「中身を全て読む」のではなく「まず `--help` で実行する」ように指示に含めると効率的です。
